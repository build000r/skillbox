## Rejected attempt and fix

An earlier direct harness invocation found that the clipboard had changed from
the known fixture. That attempt was rejected as proof, its exact remote
artifact and unsubmitted `devbox-1` attachment were removed, and its transient
Mac snapshot was deleted. The implementation was then hardened in two ways:

1. Remote smart-paste now destroys transient Mac snapshot files on success and
   every failure path.
2. Live validation can require an explicit expected SHA-256, refusing transfer
   when the current clipboard is not the intended proof artifact.

This incident is evidence for the product rule: only a real explicit gesture
authorizes ordinary use. Test automation must additionally bind itself to an
expected digest; merely invoking the dispatcher is not equivalent to user
intent.

## Not claimed in this artifact

- Automated AppleScript key injection is not counted as physical-key proof.
  macOS would focus the requested surface but the synthetic key did not reach
  even an isolated plain tmux `C-v` binding reliably.
- Other live SSH/mosh hosts and other remote tmux sessions were deliberately
  not touched after the operator restricted scope to `devbox-1`.
- Direct non-tmux `d3` and the full Sweet/Jeremy/Conference text matrix remain
  separate release gates; this artifact does not silently count them.
- Source was subsequently hardened with exact deletion of newly-created remote
  artifacts after post-transfer cancellation. That receiver revision is not
  deployed or claimed here because the operator restricted all further remote
  work to the existing `devbox-1` session and the receiver path is shared by
  the remote account.
