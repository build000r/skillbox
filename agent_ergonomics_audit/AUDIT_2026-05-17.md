## High

### H-SEC-1: SSH argv missing `--` separator on MCP path
- **Domain:** security
- **Location:** `scripts/operator_mcp_server.py:531`
- **Issue:** `cmd = ["ssh", *ssh_opts, f"{user}@{host}", command]` — no `--` separator before the destination. The recent `feat(security): validate operator identifiers and harden ssh argv` commit added the separator in `box.py:ssh_cmd()` but not here.
- **Root cause:** Hardening was applied to `box.py` only; `operator_mcp_server.py` has its own `run_ssh()` helper that didn't get the same treatment.
- **Fix:** `cmd = ["ssh", *ssh_opts, "--", f"{user}@{host}", command]`. Mirror the box.py pattern exactly.
- **Reproducer:** Craft a `boxes.json` with `tailscale_hostname: "-oProxyCommand=…"` (or any string starting with `-`); call `operator_box_exec` against it. Without `--`, ssh interprets the destination as a flag.

### H-SEC-2: MCP `run_ssh()` skips user/host validation
- **Domain:** security
- **Location:** `scripts/operator_mcp_server.py:757-769`
- **Issue:** `host` and `user` are pulled from inventory JSON and passed straight to `run_ssh()`. `box.py` defines `_validate_ssh_user` / `_validate_host` (lines 123-132) and calls them before every ssh — the MCP server doesn't import or use them.
- **Root cause:** Two parallel SSH helpers; only one was hardened.
- **Fix:** Import the validators (or re-define locally) and call them on `user`/`host` before `run_ssh()`. Return `_error_content({"type": "invalid_box_config", …})` on failure so agents see a structured error, not a stack trace.
- **Reproducer:** Edit `workspace/boxes.json` to set `ssh_user: "root; id;"` for a box, then call `operator_box_exec(box_id=…)`. With validation, the call rejects cleanly; without, the bogus value reaches the ssh argv.

### H-PERF-1: `box status` serial fleet health is O(N · 15-80s)
- **Domain:** performance
- **Location:** `scripts/box.py:3200, 3217-3276` (`cmd_status` → `box_health` per box)
- **Issue:** Listing the fleet runs `box_health()` per box sequentially. Each call: resolves ssh target (up to 3 candidates × `wait_for_ssh`), runs a remote `docker compose ps`, then 4 serial network checks. For 3 boxes this is ~60-80s of wall time on a single user-facing command.
- **Root cause:** Tight sync loop, no concurrency, network checks recomputed every call.
- **Fix:** Wrap the per-box block in `concurrent.futures.ThreadPoolExecutor(max_workers=min(5, len(boxes)))`. The work is independent and I/O-bound — straightforward parallelism.
- **Expected impact:** 3-box fleet ~80s → ~20s. Linear improvement with fleet size up to worker count.

### H-PERF-2: `wait_for_ssh` retries candidates serially with fixed backoff
- **Domain:** performance
- **Location:** `scripts/box.py:854-892` (`wait_for_ssh`, `resolve_box_ssh_target`)
- **Issue:** `resolve_box_ssh_target` tries each candidate (`tailscale_ip`, `tailscale_hostname`, `droplet_ip`) end-to-end before falling through. `wait_for_ssh` uses a fixed 1s sleep × 5 retries × 3 candidates → ~45s worst case per box.
- **Root cause:** Sequential candidate probe; no caching of last-known-good target.
- **Fix:** (a) Probe candidates in parallel and short-circuit on first-success. (b) Persist the winning target into the inventory record so the next call skips probing.
- **Expected impact:** Cold path: 45s → ~5s. Warm path (cached): near-zero.

### H-MCP-1: Dry-run marker TTL mismatch between server and guard
- **Domain:** mcp-ux
- **Location:** `scripts/operator_mcp_server.py:60` (`DRYRUN_MARKER_TTL_SECONDS = 600`) vs `scripts/guard-destructive-op.sh:283` (`MARKER_TTL_SECONDS=${SKILLBOX_DRYRUN_MARKER_TTL_SECONDS:-3600}`)
- **Issue:** Server treats the marker as valid for 10 minutes; the harness guard accepts it for 60 minutes. An agent that dry-runs at t=0 and confirms at t=11min passes the guard but is rejected by the server with `dry_run_required`. The agent has no way to tell which layer rejected it.
- **Root cause:** Two enforcement points, two hardcoded TTLs, no shared source of truth.
- **Fix:** Read `SKILLBOX_DRYRUN_MARKER_TTL_SECONDS` from env in both places with the same default (600s is safer). Add the value to the rejection message so agents see "marker expired (age=720s, ttl=600s)".

### H-MCP-2: Return shape conflates partial success with hard failure
- **Domain:** mcp-ux
- **Location:** `scripts/operator_mcp_server.py:773-798` (`handle_operator_compose_up`), `928-936` (`_ok_content`/`_error_content`)
- **Issue:** `operator_compose_up` returns `isError=true` whenever any step fails — even if `build` succeeded and only the `up-surfaces` step failed. Agents reading `isError` will assume nothing landed and may retry from scratch, redoing the build.
- **Root cause:** The error envelope is set based on `all_ok`, but the per-step structure inside `steps[]` is what actually tells the story.
- **Fix:** Either (a) make `isError` true only when the top-level intent failed (e.g., the `up` step itself), and surface partial-failure detail in `steps`, or (b) document in the description that agents should always read `steps[]` regardless of `isError`. (a) is the cleaner answer.

## Medium

### M-SEC-1: Path traversal in `load_profile()`
- **Location:** `scripts/box.py:1332-1346`
- **Issue:** `path = PROFILES_DIR / f"{name}.yaml"` followed by `path = PROFILES_DIR / name` accepts `../../../something` and resolves outside `workspace/box-profiles/`. Even with the `.yaml` suffix appended, an attacker who controls `--profile` can read any `*.yaml` file the process can see.
- **Fix:** Validate `name` against `^[a-zA-Z0-9._-]+$` before joining. Then `path.resolve().is_relative_to(PROFILES_DIR.resolve())` as belt-and-suspenders.
- **Note:** Reachable via `operator_provision(profile=…)`, so the MCP surface widens the blast radius beyond local CLI.

### M-SEC-2: Subprocess stderr forwarded verbatim into MCP responses
- **Location:** `scripts/operator_mcp_server.py:448` (`run_script`), `565` (`run_ssh`)
- **Issue:** Stderr from `box.py`, `04-reconcile.py`, and remote ssh commands is captured and returned in the MCP response. If a remote tool echoes `SKILLBOX_DO_TOKEN=…` on failure, the token lands in the agent's transcript.
- **Fix:** Filter stderr through a small redactor before returning (pattern: `(TOKEN|KEY|SECRET|PASSWORD|AUTHORIZATION)=\S+`). Also cap length at ~4KB.

### M-PERF-1: Inventory parsed per command, not per process
- **Location:** `scripts/box.py:1417-1425` (`load_inventory`)
- **Fix:** Memoize per-process; invalidate on mtime change. Most operator commands hit it 3-5x.

### M-PERF-2: MCP subprocesses re-parse `.env`/`.env.box` per tool call
- **Location:** `scripts/operator_mcp_server.py:401-413` + `436` (`run_script`)
- **Fix:** Load envs once at server startup, pass via `env=` to `subprocess.run`. Saves ~10-50ms per call; compounds on rapid sequences.

### M-MCP-1: `credential_status` is opaque inside a JSON-encoded text field
- **Location:** `scripts/operator_mcp_server.py:178-180` (description vs return wrapping)
- **Issue:** The description says "stop if `credential_status.missing` is non-empty" but the return is wrapped as `{"content":[{"type":"text","text":"<stringified json>"}]}`. Agents must double-parse to find it.
- **Fix:** When the underlying call returns structured credential info, hoist it into a top-level field on the MCP response so the agent doesn't need to JSON-parse the text field.

### M-MCP-2: `operator_compose_up.safe_first_call` points the wrong direction
- **Location:** `scripts/operator_mcp_server.py:311-313`
- **Issue:** `safe_first_call="operator_doctor"` suggests doctor before compose-up. Doctor reads state; before compose-up there's nothing to read. The agent gets a wasted call.
- **Fix:** Set `safe_first_call="operator_compose_up(build=false)"` (dry compose) and move `operator_doctor` into `next_tools`.

### M-MCP-3: Filesystem dry-run markers have no session isolation
- **Location:** `scripts/operator_mcp_server.py:889-922`
- **Issue:** Two concurrent agent sessions can race on the same marker file; Session B's dry-run can satisfy Session A's real call.
- **Fix:** Either include a `session_id` (or transport-provided client id) in the marker and check on read, or document the assumption that only one session drives destructive ops at a time.

- **Fix:** `.DEFAULT_GOAL := help` at top. One line.

### L-CLI-2: Guard error doesn't mention the MCP dry-run-marker mechanism
- **Location:** `scripts/guard-destructive-op.sh:263-276`
- **Issue:** Error text tells humans about `/commit` and `git push`, but agents driving via MCP need to know about the marker file path and the dry-run-first contract.

### L-PERF-1: No `.dockerignore`
- **Location:** repo root
- **Issue:** Build context includes `.git`, `.skillbox-state`, `logs`, `repos/*/node_modules`. Slows rebuilds and bloats image cache lookups.
- **Fix:** Add a `.dockerignore` covering those paths.

### L-INST-1: install.sh retries failed downloads with no backoff
- **Location:** `install.sh:425-438`
- **Fix:** Sleep 2s before the second attempt; cap at 3 attempts.

## Spec drafts (High findings only)

Below — one `/describe`-shaped spec per High finding. Each has at least one happy-path TC and one regression guard.

### Spec: H-SEC-1 — SSH argv hardened on MCP path

**Type:** bug-fix
**Affected:** `scripts/operator_mcp_server.py:531` (`run_ssh`)
**Current behavior:** ssh argv lacks `--` separator, allowing flag-like host strings to be interpreted as ssh options.
**Desired behavior:** ssh argv matches `box.py:ssh_cmd()` — `["ssh", *ssh_opts, "--", f"{user}@{host}", command]`.
**Root cause:** Hardening landed in `box.py` only; this parallel helper was missed.

**TC-1 (happy):** Given a box with a normal hostname, when `operator_box_exec(box_id=<id>, command='echo ok')` runs, then the call succeeds with stdout `ok`. Type: happy.
**TC-2 (regression guard):** Given a `boxes.json` entry whose `tailscale_hostname` starts with `-` (e.g., `-oProxyCommand=…`), when `operator_box_exec` is invoked against it, then ssh treats the string as a destination (not a flag), so the call fails fast with a host-resolution error rather than executing the injected option. Type: regression.

**Validation commands:**
- `python3 -c "import scripts.operator_mcp_server as m; import inspect; assert '\"--\"' in inspect.getsource(m.run_ssh) or \"'--'\" in inspect.getsource(m.run_ssh)"`
- Existing test suite: `make test`

### Spec: H-SEC-2 — MCP `run_ssh` validates user and host

**Type:** bug-fix
**Affected:** `scripts/operator_mcp_server.py:757-769`
**Current behavior:** `user`/`host` from inventory go straight to `run_ssh` with no shape check.
**Desired behavior:** Both are validated against the same regexes used in `box.py` (`_SSH_USER_RE`, `_HOST_RE`). On failure, return `_error_content({"error":{"type":"invalid_box_config", …}})`.

**TC-1 (happy):** Given a normal inventory entry, when `operator_box_exec` runs, then ssh is invoked as today.
**TC-2 (regression guard):** Given an inventory entry with `ssh_user: "root; id;"`, when `operator_box_exec` is called, then the response is an `invalid_box_config` error and no subprocess is spawned.
**TC-3 (consistency):** The validators used here are the same module/regex as in `box.py` — if a value is rejected by one path it is rejected by the other.

**Validation commands:**
- Unit test: feed a crafted inventory dict into `handle_operator_box_exec`; assert `invalid_box_config`.
- `grep -n '_validate_ssh_user\|_validate_host' scripts/operator_mcp_server.py` returns matches around line 750.

### Spec: H-PERF-1 — Parallel fleet health in `box status`

**Type:** performance
**Affected:** `scripts/box.py:3200` (`cmd_status`)
**Current behavior:** `box_health` runs sequentially per box; 3 boxes = ~60-80s.
**Desired behavior:** `box_health` per box runs concurrently via `ThreadPoolExecutor(max_workers=min(5, N))`. Output order matches inventory order. JSON shape unchanged.

**TC-1 (happy):** Given a 3-box inventory of reachable boxes, when `box status` runs, then wall time < 25s on a healthy network. Type: happy.
**TC-2 (regression guard):** Output JSON for each box is byte-identical to the previous serial implementation (snapshot test on a stub inventory with mocked ssh).
**TC-3 (error containment):** If one box ssh hangs to timeout, the other boxes still report — total time bounded by `timeout + ε`, not `N × timeout`.

**Validation commands:**
- `time python3 scripts/box.py status --format json` against a known fleet — compare to baseline.

### Spec: H-PERF-2 — Parallel SSH candidate probing + cache

**Type:** performance
**Affected:** `scripts/box.py:854-892` (`wait_for_ssh`, `resolve_box_ssh_target`)
**Current behavior:** Candidates probed serially, 5 retries × 1s each, 3 candidates → 45s worst case.
**Desired behavior:** Probe all candidates in parallel, return first success. Persist winning target to inventory; on the next call, try cached target first.

**TC-1 (happy):** Given a box with one reachable candidate, when `resolve_box_ssh_target` runs cold, then it returns in ≤ probe-timeout + 1s.
**TC-2 (cache):** Given a recent successful probe, when the same box is queried again, then the cached target is used without a probe round-trip.
**TC-3 (regression):** If all candidates fail, the function still returns `None` and emits the same error string as today.

**Validation commands:**
- Unit test with monkeypatched socket connect; assert wall-time < 3s with parallel probes vs. serial baseline.

### Spec: H-MCP-1 — Unified dry-run marker TTL

**Type:** bug-fix
**Affected:** `scripts/operator_mcp_server.py:60`, `scripts/guard-destructive-op.sh:283`
**Current behavior:** Server TTL = 600s; guard TTL = 3600s. Mismatch causes confusing rejections.
**Desired behavior:** Both read `SKILLBOX_DRYRUN_MARKER_TTL_SECONDS` from env; both default to 600s. Rejection message includes observed age and configured ttl.

**TC-1 (happy):** With default TTL, a dry-run at t=0 followed by a real call at t=300s succeeds.
**TC-2 (regression guard):** A dry-run at t=0 followed by a real call at t=700s is rejected by *both* server and guard (no path that lets one through and the other through).
**TC-3 (env override):** Setting `SKILLBOX_DRYRUN_MARKER_TTL_SECONDS=120` changes both layers' threshold.

**Validation commands:**
- Script: `dry_run` then sleep 700s then real — observe consistent rejection.

### Spec: H-MCP-2 — `isError` reflects top-level intent only

**Type:** bug-fix
**Affected:** `scripts/operator_mcp_server.py:773-798` (`handle_operator_compose_up`)
**Current behavior:** `isError=true` whenever any step fails, even if the primary step succeeded.
**Desired behavior:** `isError` reflects the success of the headline step (`up`). Per-step success/failure is in `steps[]`. Description updated to instruct agents to read `steps[]` for partial-failure detail.

**TC-1 (happy):** All three steps succeed → `isError` absent/false; `steps` all `ok=true`.
**TC-2 (partial):** `build` ok, `up` ok, `up-surfaces` fails → `isError=false` (top-level intent landed), `steps[2].ok=false`, `next_actions` empty.
**TC-3 (hard fail):** `up` fails → `isError=true`, `steps[…].ok=false` for that step.

**Validation commands:**
- Mock `run_compose` to force the matrix of outcomes; assert response shape per TC.
